# fix-79c0f82dd1ad7917 — Time-limited messages lose Redis TTL after read, causing storage leak

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 EPTX

In `retrieval.ts:54-55`, when a time-limited message is read before expiry, `message.readCount` is incremented and `store.save(message)` is called **without a TTL**. The Redis `SET` command without `EX` removes any previously-set expiry, making the key persistent. If the message is never accessed again after its logical expiry (`expiresAt`), it permanently leaks in Redis. The `save` call should either re-compute and pass the remaining TTL, or use Redis `KEEPTTL` to preserve the original expiry.
