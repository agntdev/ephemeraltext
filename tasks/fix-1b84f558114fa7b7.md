# fix-1b84f558114fa7b7 — Time-limited messages lose Redis TTL after first read, leaking storage

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 EPTX

In `retrieval.ts:55`, time-limited messages are re-saved via `await store.save(message)` with **no TTL argument** after each successful read. The initial save in `bot.ts:259` passes `ttlSeconds` (e.g. `Math.ceil((expiresAt - now) / 1000)`), which sets a Redis `EX` expiry. The re-save calls `SET key value` without `EX`, stripping the TTL entirely.  
After the first read, the key persists in Redis indefinitely — it is only cleaned up if someone later attempts to read it after `expiresAt` (triggering the manual `store.delete` in `retrieval.ts:32`). If nobody accesses the expired message, the ciphertext leaks permanently. Pass the remaining TTL (`Math.ceil((message.expiresAt - now) / 1000)`) on re-save.
