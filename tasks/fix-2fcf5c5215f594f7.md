# fix-2fcf5c5215f594f7 — No rate limiting on admin commands

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 EPTX

The `/admin` command chain has no rate limiter, unlike the `/upload` flow which enforces a 10-per-hour cap. Repeatedly calling `/admin metrics` triggers a full Redis `SCAN` over all message keys, and `/admin logs` adds a new audit entry on each invocation. An authenticated admin (or a compromised admin credential) could use this to degrade performance or flood the audit log.
